## Sprint Review
### Завершенные задачи
1. **Создание новой задачи** — реализована форма с валидацией и сохранением в базу данных.
2. **Отображение задач на Kanban-доске** — задачи сортируются по статусам (To Do, In Progress, Done).
3. **Реализация темной темы**

### Что было доставлено по итогу спринта
В результате спринта пользователи могут:
1. Создавать новые задачи с указанием названия, описания, приоритета и исполнителя
2. Видеть все задачи на интерактивной Kanban-доске
3. Перетаскивать задачи между колонками (смена статуса)
4. Видеть уведомление об успешном создании задачи

### Незавершенные задачи
**Редактирование существующих задач** — не удалось завершить из-за сложностей с формой редактирования и синхронизацией данных. Задача переносится в следующий спринт.
**Добавление drag-and-drop для перемещения задач** — осталось проверить на наличие багов

## Sprint Retrospective
### Что прошло хорошо
1. Четкое распределение задач по приоритетам
2. Быстрое решение технических проблем благодаря ежедневным стендапам
3. Хорошая коммуникация в команде через Discord

### Что пошло не так
1. Недооценили сложность задачи по редактированию задач
2. Возникли проблемы с тестированием drag-and-drop функционала
3. Не хватило времени на рефакторинг кода

### Что можно улучшить в следующем спринте
1. Проводить более детальный анализ сложности задач перед планированием
2. Выделять время на технический долг и рефакторинг
3. Улучшить процесс код-ревью
4. Добавить больше автоматизированных тестов
